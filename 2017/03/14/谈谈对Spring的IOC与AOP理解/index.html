谈谈对Spring的IOC与AOP理解

Spring
  • 总结

前言

用了SpringMVC框架这么久，结合一些网上的说法，谈谈自己对Spring的IOC与AOP的理解。

IOC

概念

IOC(Inversion of Controller)：有三种说法控制反转，依赖注入，依赖倒置，而所谓的IOC对于Spring来说就是负责控制对象的生命周期和对象间的关系。

个人理解

打个比方以前我们吃饭都需要去亲自去快餐店或者其他小餐馆去吃饭，还要看下距离远不远，或者其他因素条件。现在有许多的人都开始在美团点外卖了，只需要在美团上注册账号填好地址，点自己想吃的，那么你不需要关心别人是怎么送过来的，有多远，那么美团这个平台会负责帮你把外卖送到你手上。其中美团平台为Spring，它负责帮你把你所需要的东西自动送上门。比如Spring中的Controller与Service，Controller只管业务逻辑，不需要管这个业务做的过程，Spring通过@Resource把你所需涉及的业务(Service)注入到Controller，这也就是依赖注入。Spring依赖注入的实现技术就是反射机制。

AOP

概念

AOP(Aspect Oriented Programming)：面向切面编程，java是OOP语言，是一种纵向的扩展，而AOP就是将横向与业务无关的代码逻辑封装起来，目标就是分离关注点。Spring声明事务就用了AOP，在访问数据库时，对于事务的处理与回滚，Spring会主动帮你完成，而你只需要访问数据库。

个人理解

打个比方你和一个美女(帅哥)约会吃饭，出门前，你是不是要想自己打扮的怎么样，穿着怎么样，带够了钱没，然后再出去，而AOP的思想就是，你的打扮有专业的化妆师，穿着有设计师，钱有人帮你付，这些都不需要你考虑，你只需要去吃个饭就行了，这就是AOP思想。

结语
